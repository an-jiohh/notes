# 캐시 추상화


스프링은 추상화를 통해 다양한 기술들을 제공하고 있음  
데이터베이스의 접근의 경우 JDBC, JPA에 종속적이지 않고 추상화되어 사용할 수 있음

캐시 역시 스프링에서 추상화를 제공함  
특정 캐시 구현체 (예: Caffeine, Redis 등)에 종속되지 않고 유연한 설계가 가능함

---
## Cache 인터페이스

| **메서드**                                            | **설명**                              |
| -------------------------------------------------- | ----------------------------------- |
| String getName()                                   | 이 캐시의 이름을 반환                        |
| ValueWrapper get(Object key)                       | 키에 연결된 값을 가져온다 (없으면 null 반환 가능)     |
| void put(Object key, Object value)                 | 키-값을 캐시에 저장                         |
| ValueWrapper putIfAbsent(Object key, Object value) | 키에 값이 없을 때만 저장하고, 기존 값이 있으면 그 값을 반환 |
| void evict(Object key)                             | 특정 키를 캐시에서 제거                       |
| void clear()                                       | 전체 캐시를 비움                           |

기본적으로 캐시 구현체는 캐시이름을 가지며 이를 통해 접근 및 사용된다.
